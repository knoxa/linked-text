Rewriting claims

This is an example of rewriting claims to resolve entity references.

Suet. Cal. 7:

Germanicus married Agrippina, the daughter of Marcus Agrippa and Julia, by whom he had nine children, two of whom died in their infancy, and another a few years after; ...

We make a claim ...

  • Agrippina is the daughter of Julia.

Julia is ambiguous, so we create a rewrite rule:

"Julia" refers to Julia the Elder.

... and, at the same time, add content="Julia the Elder" to the HTML span element containing "Julia" in the original argument (see the source for this page).

An XSLT transform takes this page and generates an AIF argument map. This will extract both the claim as presented (Agrippina is the daughter of Julia) and the rewritten claim (Agrippina is the daughter of Julia the Elder), and link the two via an AIF MA-Node.

The unambiguous name (underlined above) is inside a span element with class="name". This makes it possible to find any rewrite rules that justify the rewritten claim and add them as premises.
